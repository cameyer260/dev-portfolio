# Examples Design Notes

This file applies to the `/examples` system only. It does not apply to the main portfolio homepage.

## Permanent Guidance

- Keep `/examples` separate from the main portfolio design language
- Keep shared infrastructure, shared content schema, theme tokens, and shared primitives
- Keep per-industry renderers responsible for page composition so the pages do not feel cloned
- Prefer believable photography or a visible placeholder over invented artwork
- Keep mobile conversion strong and keep the fictional-business disclosure visible on every detail page

## Avoid

- Reintroducing one rigid shared page template for every industry
- Adding global main-portfolio rules just to support `/examples`
- Letting new abstractions flatten the differences between industries
